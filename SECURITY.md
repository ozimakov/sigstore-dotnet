# Security Policy

## Supported versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Reporting a vulnerability

**Please do not file public GitHub issues for security vulnerabilities.**

Report suspected security issues by email to the repository maintainers (see GitHub profile contact for `ozimakov` or use the address published on the repository once available). Include:

- A clear description and impact assessment
- Steps to reproduce (proof-of-concept, if possible)
- Affected versions or commit hashes

## Response timeline

- **Acknowledgement:** within 48 hours of report receipt (business days).
- **Remediation:** we aim to provide a fix or mitigation plan within 90 days for issues confirmed as valid, depending on severity and coordination with upstream Sigstore components.

## CVE assignment

Where appropriate, issues will be coordinated for CVE assignment through GitHub security advisories and/or the MITRE/CVE process, in line with community practice.
